# 커피점 & 제과점 거리 합 최적화

## 1. 문제 개요

$N$개의 건물과 $K$개의 양방향 도로가 주어질 때, 특정 조건을 만족하는 **주택(House)** 중 거리 합의 최솟값을 구하는 문제.

* **주택의 조건**: 주어진 커피점($M$개) 및 제과점($P$개) 목록에 포함되지 않은 건물.
* **제약 조건**: 커피점과의 거리 $\le R$ **AND** 제과점과의 거리 $\le R$.
* **최적화 목표**: `(가장 가까운 커피점 거리) + (가장 가까운 제과점 거리)`의 최솟값.

## 2. 핵심 설계 전략

### 🚀 다중 시작점 다익스트라 (Multi-source Dijkstra)

모든 커피점과 제과점의 조합을 개별적으로 계산하면 $M \times P \times \text{Dijkstra}$로 인해 시간 초과가 발생한다. 이를 해결하기 위해 **BFS의 다중 시작점 원리**를 다익스트라에 적용한다.

1. **커피점 그룹**: 모든 커피점을 우선순위 큐(PQ)에 거리 `0`으로 넣고 시작. → 한 번의 연산으로 **모든 건물에서 가장 가까운 커피점까지의 거리** 도출.
2. **제과점 그룹**: 모든 제과점을 PQ에 거리 `0`으로 넣고 시작. → 한 번의 연산으로 **모든 건물에서 가장 가까운 제과점까지의 거리** 도출.

### 🏠 주택 판별 (House Identification)

`calculate` 함수가 호출될 때마다 커피점과 제과점 구성이 달라지므로, 효율적인 판별이 필요하다.

* **Marking Array**: 호출 시마다 `boolean` 또는 `int` 배열을 사용하여 커피점/제과점 ID를 마킹한다.
* 마킹되지 않은 인덱스($i$)에 대해서만 거리 합을 계산한다.

## 3. 알고리즘 프로세스

1. **초기화 (`init`, `add`)**:
* 도로 정보를 **양방향 인접 리스트**로 구성.


2. **최단 거리 계산 (`calculate`)**:
* `distCoffee[]`: 모든 커피점을 시작점으로 다중 시작점 다익스트라 수행.
* `distBakery[]`: 모든 제과점을 시작점으로 다중 시작점 다익스트라 수행.


3. **최솟값 갱신**:
* $0$부터 $N-1$까지 순회하며:
* `if (isHouse[i] && distCoffee[i] <= R && distBakery[i] <= R)`
* `min_dist = min(min_dist, distCoffee[i] + distBakery[i])`





## 4. 복잡도 분석

* **시간 복잡도**: $O(T \times (\text{calculate 횟수} \times E \log V))$
* $V=10,000, E=30,000$, `calculate`=100회
* 약 $100 \times 30,000 \times 14 \approx 4.2 \times 10^7$ 연산으로 제한 시간(1~6초) 내 충분히 통과 가능.


* **공간 복잡도**: $O(V + E)$ (인접 리스트 및 거리 배열 저장).
